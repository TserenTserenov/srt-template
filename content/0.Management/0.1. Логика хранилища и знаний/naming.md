# Стандарт именования: S2R / SRR

## Каноническая схема имён

Для предотвращения дрейфа терминологии (SRT/SRD/SRR и т.п.) фиксируем следующие имена:

### Иерархия имён

| Уровень | Имя | Назначение |
|---------|-----|------------|
| **Бренд** | **S2R** | Короткое имя метода, произносится вслух, пишется крупно |
| **Формальная расшифровка** | **SRR — Systems–Roles Repository** | Точное определение для документации |
| **RU-эквивалент** | **Хранилище систем и ролей** | Русская расшифровка |
| **Имя шаблона** | **S2R Template** | Этот репозиторий (референсная реализация) |

### Каноническая формула

> **S2R Template** — шаблон репозитория для **SRR (Systems–Roles Repository)**: хранилища проекта по матрице *Системы × Роли* (3×3) с опорой на FPF.

---

## Устойчивые формулировки

### Как говорить об этом

| Контекст | Формулировка |
|----------|--------------|
| Коротко | «Мы ведём проект в S2R» |
| Точнее | «S2R — это SRR: Systems–Roles Repository» |
| Про репозиторий | «Это S2R Template — стартовый шаблон SRR под GitHub» |

### Связь S2R и SRR

Одно — **бренд**, второе — **расшифровка**. Нет расхождения между ними:

- **S2R** = бренд/ярлык (как Docker, не "Containerized Application Runtime")
- **SRR** = формальное имя (Systems–Roles Repository)

## Мини-стандарт терминов

Зафиксированная терминология (не меняем):

| Термин | Паттерн | Описание |
|--------|---------|----------|
| **Matrix / матрица** | — | Модель 3×3 (системы × роли) |
| **Families / семейства** | F0-F9 | Наборы документов по ячейкам матрицы |
| **Management family / управляющее семейство** | F0 | Правила ведения хранилища |

---

## Критически важно

**S2R/SRR — это метод организации хранилища**, а не "процесс разработки".

- Процессы описываются **внутри** ячеек матрицы
- Метод остаётся **структурой знаний**
- S2R не является заменой архитектурных фреймворков, а способом разложить их результаты по адресам
