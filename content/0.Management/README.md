# Раздел 0. Управление (Management)

## Назначение раздела

Раздел **Management** — это **метасистема хранилища**, которая управляет логикой организации знаний, рабочими процессами и автоматизацией. Это "пульт управления" всем репозиторием.

В терминах FPF (паттерн A.1 — Holonic Foundation): Management — это холон, который одновременно является частью большей системы (проекта) и содержит свои подсистемы (логику, архитектуру, операции).

## Обзор раздела

Management выполняет пять ключевых функций:

| Функция | Описание | Подраздел |
|---------|----------|-----------|
| **Логика хранилища** | Принципы организации, терминология, структура | 0.1.Meaning |
| **Архитектура управления** | Роли, ответственность, оргструктура | 0.2.Architecture |
| **Процессы работы** | Workflows, создание документов, интеграция с AI | 0.3.Operations |

## Принципы работы с разделом

### 1. SSOT (Single Source of Truth)
Каждая сущность имеет одно главное место определения. Дублирование — через ссылки, не копирование.

### 2. Метаданные обязательны
Все документы в `content/` должны иметь YAML frontmatter с обязательными полями.

### 3. Разделение ролей
- **Люди** редактируют стратегию и архитектуру
- **AI** создаёт производные артефакты и помогает с рутиной

### 4. Инкрементальное развитие
Структура развивается по мере необходимости. Не создавай пустые документы "на будущее".

## FPF-интеграция

Раздел Management использует следующие паттерны FPF:

| Паттерн | Применение |
|---------|------------|
| **A.1 Holonic Foundation** | Каждый документ — холон в иерархии |
| **A.1.1 Bounded Context** | Каждая папка — ограниченный контекст |
| **A.7 Strict Distinction** | Чёткое различение: документ ≠ процесс |
| **B.3 Trust Calculus** | F-G-R метрики для оценки документов |
